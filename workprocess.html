Arbetsprocess

Process

Först gjorde jag en mapp för Moment 1, därefter öppnade jag mappen i VSC och vidare till terminalen och skrev npm init. Jag glömde skriva -y så jag blev lite förvirrad, men eftersom jag har gjort liknande förut så klickade det ganska snabbt och jag började spamma enter.

Efter det skapade jag filstrukturen med bilder, sidor, skript och stilar. När det var klart ersatte jag "main" i package.json till "source" med en filväg till index.html. Därefter installerade jag Parcel och skrev i "start": "parcel" och "build": "parcel build" i "scripts."

Jag vet inte när, men jag gjorde i början en branch kallad "Dev." I den gjorde jag alla ändringar och direkt mergade in den med main för att se ifall sidan på GitHub Pages uppdaterades. Vilket det gjorde så allt fungerade då, men den exekverade inte build parcel vilket upptäcktes vid när photos.html gjordes. Eftersom jag inte hittade hur man fixade det till GitHub Pages i teori materialet frågade jag ChatGPT och då fick jag bra instruktioner på hur jag skulle göra det.

Instruktionerna var att man behövde göra någon slags deploy.yml i .github/workflows och automatiskt publicera till gh-pages branchen från main branchen. Jag förstår inte riktigt än hur deploy.yml fungerar, men det kan jag nog lära mig. gh-pages installerades genom npm install --save-dev gh-pages.

I början var det svårt att veta vilka filer och mappar skulle vara i .gitignore vilket orsakade att jag behövde ta bort grejer vilket var jätte svårt. Det är därför package-lock.json är kvar i GitHub för det blev en konflikt och jag råkade skriva över när jag inte skulle det. Orkade inte fixa det.

När photos.html bildernas automatiska bildkonvertering till avif äntligen fungerade i GitHub Pages började jag testa med sharp.config.json med att få ner kvalitén men ändå se bra ut.

Det var ett fel som jag inte visste vars det var som gjorde att jag behövde göra jätte många commits. Dessa commits var enbart jag som försökte fixa filvägarna som egentligen inte var felet utan att det var något fel med dist/. Jag vet inte om det är fixat. Jag tror mycket av felen var att jag gjorde fel i början, det var mitt fel att inte testa innan jag började med laborationen. Jag trodde det skulle gå bra, ska repetera från npm init till deploy gh-pages så att jag har det inlärt.

Uppgiften var kul trots att jag nog krånglade till det.
